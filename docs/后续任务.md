# 后续任务清单

## 1. 测试体系强化
- **扩充真实单测**：目前仅有 `tests/sanity.test.ts` 作为占位，需要针对关键模块（如商家跳转、Quota 管理、批量转录、受众分析 Hooks）编写实际测试用例。
- **完善测试初始化**：根据真实测试需求在 `tests/setup.ts` 中加入必要的 polyfill、mock 或 Testing Library 配置，并保持与 Vitest 配置同步。

## 2. 导航/CTA 配置统一
- **抽离路由配置**：将首页、Header、Footer 等散落的 `Link` 地址集中到单一配置文件，避免再次出现指向不存在路由的硬编码。
- **建立自动校验**：可通过脚本或单测遍历配置中的路径，确保它们都能在 `app/` 目录中找到对应页面。

## 3. Quota 管理与日志细化
- **监控/告警**：为 `QuotaManager` 中的关键 `console` 警告接入统一日志系统（如 Sentry/Elastic），便于观察月度重置、限额告警。
- **批量重置任务**：检查现有 Cron/脚本是否已调用 `resetAllUsersMonthlyUsage`，若无，需要补充部署方案与文档。

## 4. 页面内容与 UX
- **补全 `Documents` 与 `Feedback` 内容**：当前为静态占位，可逐步填入真实文档链接、提交流程、客服联系方式。
- **受众分析操作提示**：在 `AudienceAnalysisSection` 中为非管理员用户提供明确指引（如何申请权限/查看日志）。

## 5. 自动化校验
- **CI 脚本**：将 `pnpm lint && pnpm type-check && pnpm test:run` 纳入 CI 流水线，确保每次提交都触发同样校验。
- **E2E 计划**：根据 `project-doc` 要求，评估是否需要为新增路由补齐 Playwright 场景，避免回归。
